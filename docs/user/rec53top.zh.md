# rec53top 概览

`rec53top` 是 rec53 的本地终端面板。它直接读取一个 rec53 的 Prometheus 指标端点，并把当前进程状态整理成固定的六宫格视图，分别展示流量、缓存、snapshot、上游、XDP 和状态机。

## 用途

`rec53top` 适合在还没有完整 Prometheus/Grafana 流程时，先快速看本地信号。

典型场景：

- 开发机或实验机上的第一轮排查
- 配置变更、重启、升级后的本地验证
- 在一个终端里快速比较“当前短窗口变化”和“启动以来累计值”
- 当你想先看哪些代码路径计数在增长时，不必立刻切原始指标页面

## 如何阅读

概览页回答第一个问题：现在哪个区域最可疑。

详情页回答下一个问题：当前短窗口里哪些信号最突出、哪些累计计数在增长、下一步该看哪里。

TUI 故意把这两类视图分开：

- `Current window` 看最近速率、比例和当前最突出的问题
- `Since start counters` 看有界累计总量和 top-N 热路径

这样更容易判断：问题是还在发生，还是主要只是历史累计。

## 当前边界

`rec53top` 在 `v1.1.x` 里故意保持很窄：

- 只看单个目标
- 只在本地终端运行
- 只读面板
- 不做持久历史存储
- 不做多节点/集群视图
- 不做多级树状钻取，只保留一个详情页和少量子视图

它的作用是缩小排查范围，不替代原始 `/metric`、日志、Prometheus 查询或未来的多节点可观测性。

如果 TUI 显示了一个短趋势提示，把它当作“当前会话里这个信号是在升高还是在降温”的本地提示。历史监控还是用 Prometheus/Grafana。

## 推荐路径

使用 `rec53top` 时建议按这个顺序：

1. 先打开概览页，找出第一个可疑面板。
2. 按 `Enter` 或 `1` 到 `6` 进入详情。
3. 先看 `Summary`，如果当前面板支持，再切到钻取子视图。
4. 对比 `What stands out now` 和相关的有界总量/细分页。
5. 如果两者指向同一条路径，再去看日志或原始指标，搜索范围会小很多。
6. 如果两者不一致，通常是预热噪声、旧累计，或者一个可能已经开始衰减的回归。
