# Bug Record: 2026.3.9.001

## Original Issue

E2E tests failing with errors:
- `dns: overflow unpacking uint16`
- `dns: overflowing header size`
- `dns: buffer size too small`
- Test timeouts (2 minutes)

## Root Causes Identified

1. **CNAME Loop** - State machine had no protection against CNAME loops
2. **No Iteration Limit** - State machine could loop infinitely
3. **Cache Message Sharing** - Cached messages were shared by reference, causing corruption
4. **Missing EDNS Support** - DNS client couldn't handle large responses

## Fixes Applied

### 1. CNAME Cycle Detection (`server/state_machine.go`)
```go
// Track visited domains for CNAME cycle detection
visitedDomains := make(map[string]bool)
...
if visitedDomains[cnameTarget] {
    return nil, fmt.Errorf("CNAME cycle detected: %s", cnameTarget)
}
visitedDomains[cnameTarget] = true
```

### 2. Iteration Limit (`server/state_machine.go`)
```go
const MaxIterations = 50
...
if iterations > MaxIterations {
    return nil, fmt.Errorf("max iterations exceeded, possible CNAME loop")
}
```

### 3. Cache Message Copy (`server/cache.go`)
```go
func getCacheCopy(key string) (*dns.Msg, bool) {
    msg, found := getCache(key)
    if !found {
        return nil, false
    }
    return msg.Copy(), true
}

func setCacheCopy(key string, value *dns.Msg, expire uint32) {
    setCache(key, value.Copy(), expire)
}
```

### 4. EDNS Support (`server/state_define.go`)
```go
newQuery.SetEdns0(4096, false)
...
dnsClient := &dns.Client{
    UDPSize: 4096,
}
```

### 5. Response Initialization (`server/server.go`)
```go
result, err := Change(stm)
if err != nil {
    reply.SetRcode(r, dns.RcodeServerFailure)
} else {
    reply = result
}
```

## Remaining Issues

Some E2E tests still fail with:
- `dns: overflow unpacking uint16` - For non-existent domains and large record types (MX, TXT)
- `dns: buffer size too small` - For certain responses

These appear to be related to:
1. Test client configuration (needs EDNS)
2. Non-existent domain handling
3. Network conditions

## Status

- **Fixed**: Infinite loop, CNAME cycle, cache corruption
- **Partial**: Some E2E tests still fail due to test client configuration

## Recommendation

1. Update E2E tests to use EDNS-enabled client
2. Add better error handling for non-existent domains
3. Consider adding TCP fallback for truncated responses
